# 01 Environment Setup

## 1. What This Module Is

This module prepares your computer for Python, API, GenAI, FastAPI, RAG, MCP, LangGraph, Docker, and POC work.

It focuses on practical setup, not theory.

## 2. Why It Matters

Most beginner failures happen before coding:

- Python command not found
- package installed in the wrong environment
- VS Code using the wrong interpreter
- API key pasted directly into code
- terminal command copied without understanding
- app running but browser opened on the wrong URL

This module prevents those problems.

## 3. What You Should Finish Knowing

After this module, you should be able to:

- check whether Python is installed
- use PowerShell for basic commands
- create and activate a virtual environment
- install packages with `pip`
- freeze dependencies into `requirements.txt`
- create a `.env` file safely
- run Python files
- understand local app URLs such as `http://127.0.0.1:8000`
- troubleshoot common setup errors

## 6. Practical Scope

This module is local development focused. You are preparing your own laptop/workstation.

Production deployment is not covered here. Docker and Kubernetes come later.

## 7. What Not To Over-Focus On

Do not over-focus on memorizing every terminal command. Focus on:

- what folder you are in
- what environment is active
- what command is running
- where packages are installed
- where secrets are stored

## 8. How This Helps In AI Engineer JRS / Mettl / POC / Interview

- **JRS labs**: you can follow setup instructions without blocking.
- **Mettl**: you understand Python and package commands.
- **POC**: you can run backend and demo apps locally.
- **Interview**: you can explain virtual environments, dependencies, and API key safety.
