# 26 Practice Tracker

## 1. What this module is

This folder helps track study progress, lab completion, interview readiness, and POC readiness.

It turns the course into a practical self-study plan instead of a pile of notes.

## 2. The most important beginner idea

Mark progress honestly.

Do not mark a topic complete just because you read it once.

A topic is closer to complete when you can:

- explain it in simple words
- run a small example
- identify common mistakes
- answer interview-style questions
- connect it to the final POC

## 3. Why it matters

Beginner learners often feel they "covered" a topic but cannot use it under pressure.

The tracker forces active learning:

```text
read -> run -> explain -> practice -> self-check -> revise
```

## 4. File list

- `daily-study-plan.md`: daily routine for steady progress.
- `7-day-crash-plan.md`: compressed revision plan.
- `14-day-plan.md`: medium-speed preparation plan.
- `21-day-plan.md`: more comfortable preparation plan.
- `topic-completion-checklist.md`: concept-by-concept completion tracking.
- `lab-completion-checklist.md`: hands-on lab tracking.
- `interview-readiness-checklist.md`: speaking and Q&A readiness.
- `poc-readiness-checklist.md`: final project demo readiness.

## 5. Practical scope

Use this module every study day.

At the end of each session, update:

- what you studied
- what you ran
- what failed
- what you still cannot explain
- what needs revision

## 6. What not to over-focus on

Do not spend more time formatting the tracker than studying.

The tracker is useful only if it exposes gaps and drives practice.

## 7. How this helps in AI Engineer JRSS / Mettl / POC / interview

It ensures you are not only reading notes but also becoming ready for:

- programming assessment
- SME-led labs
- Mettl screening
- POC demo
- technical interview discussion
- stand-and-deliver explanation
