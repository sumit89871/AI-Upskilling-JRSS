# Local Models Interview Questions

## Local vs hosted model?

Short answer:

A hosted model runs on provider infrastructure. A local model runs on your machine or server.

Expanded explanation:

Hosted models are easier and often stronger, but require API keys and data review. Local models give more control but need hardware and may be slower.

Real project example:

The POC can use mock mode first, optional OpenAI/Gemini hosted mode, and optional Ollama local mode.

Common wrong answer:

"Local models are always better."

When to say this in interview:

Use when discussing model deployment tradeoffs.

## What is Ollama?

Short answer:

Ollama is a tool for running local models more easily.

Expanded explanation:

It lets you pull models and run them locally through a CLI or local API-style interface.

Real project example:

Use Ollama to test a local model for a QA assistant demo.

Common wrong answer:

"Ollama is a cloud provider only."

When to say this in interview:

Use when asked about local model usage.

## What is Hugging Face?

Short answer:

Hugging Face is an ecosystem and hub for models, datasets, tokenizers, and ML tools.

Expanded explanation:

It provides model cards, transformers library, pipelines, and community models.

Real project example:

Check a model card before using a model locally.

Common wrong answer:

"Every Hugging Face model is safe for enterprise use."

When to say this in interview:

Use when asked open model awareness.
