# FastAPI Cheatsheet

## Install Dependencies

Command:

```powershell
pip install fastapi uvicorn pydantic
```

Meaning:

Installs the FastAPI framework, Uvicorn server, and Pydantic validation library.

When to use:

Use inside an activated virtual environment when starting or running a FastAPI project.

Example:

```powershell
.\.venv\Scripts\Activate.ps1
pip install fastapi uvicorn pydantic
```

Be careful:

If the virtual environment is not activated, packages may install into a different Python environment.

## Install From Requirements

Command:

```powershell
pip install -r requirements.txt
```

Meaning:

Installs every package listed in `requirements.txt`.

When to use:

Use when a project already provides a dependency file.

Example:

```powershell
cd 06-fastapi\implementation\fastapi-ai-helper-api
pip install -r requirements.txt
```

Be careful:

Run from the folder where `requirements.txt` exists, or provide the correct path.

## Minimal FastAPI App

Syntax:

```python
from fastapi import FastAPI

app = FastAPI()
```

Meaning:

Imports FastAPI and creates the application object.

When to use:

Use at the start of a FastAPI `main.py`.

Example:

```python
app = FastAPI(title="FastAPI AI Helper API")
```

Be careful:

The object name must match the Uvicorn command. `app = FastAPI()` pairs with `uvicorn main:app --reload`.

## GET Route

Syntax:

```python
@app.get("/health")
def health():
    return {"status": "ok"}
```

Meaning:

Runs `health()` when a GET request reaches `/health`.

When to use:

Use for reading data or health checks.

Example:

```text
GET http://127.0.0.1:8000/health
```

Be careful:

The `@` is required. Without it, the function is not registered as a route.

## POST Route With Request Body

Syntax:

```python
from pydantic import BaseModel

class TextRequest(BaseModel):
    text: str

@app.post("/summarize")
def summarize(request: TextRequest):
    return {"summary": request.text[:120]}
```

Meaning:

Accepts a JSON body, validates it with Pydantic, and returns a JSON response.

When to use:

Use when the client sends structured data for processing.

Example request:

```json
{"text": "FastAPI is useful for AI APIs."}
```

Be careful:

Use `request.text`, not just `text`, because the field is inside the validated request object.

## Run App

Command:

```powershell
uvicorn main:app --reload
```

Meaning:

Runs the `app` object from `main.py` with Uvicorn.

When to use:

Use during local development.

Example:

```powershell
cd 06-fastapi\implementation\fastapi-ai-helper-api
uvicorn main:app --reload
```

Be careful:

Run from the folder containing `main.py`, unless you use a package path.

## `main:app`

Syntax:

```text
main:app
```

Meaning:

`main` means `main.py`; `app` means the FastAPI object inside that file.

When to use:

Use as part of the Uvicorn command.

Example:

```powershell
uvicorn main:app --reload
```

Be careful:

If the file is `server.py`, use `server:app`. If the object is `api`, use `main:api`.

## `--reload`

Syntax:

```powershell
uvicorn main:app --reload
```

Meaning:

Restarts the server automatically when code changes.

When to use:

Use while developing locally.

Example:

```text
Save main.py -> Uvicorn reloads -> test endpoint again
```

Be careful:

A syntax error can prevent the app from reloading until the code is fixed.

## Swagger UI

URL:

```text
http://127.0.0.1:8000/docs
```

Meaning:

Interactive API documentation generated by FastAPI.

When to use:

Use to inspect and test endpoints without writing a separate client.

Example:

```text
Open /docs -> choose POST /summarize -> Try it out -> Execute
```

Be careful:

Swagger UI shows what FastAPI understands from your routes and Pydantic models. If a schema looks wrong, check your function parameters and model definitions.

## Path Parameter

Syntax:

```python
@app.get("/requirements/{requirement_id}")
def get_requirement(requirement_id: int):
    return {"requirement_id": requirement_id}
```

Meaning:

Reads a value from the URL path.

When to use:

Use when identifying a specific item.

Example:

```text
GET /requirements/10
```

Be careful:

The name inside `{}` should match the function parameter name.

## Query Parameter

Syntax:

```python
@app.get("/search")
def search(topic: str = ""):
    return {"topic": topic}
```

Meaning:

Reads a value from the URL query string after `?`.

When to use:

Use for filters, search terms, options, and defaults.

Example:

```text
GET /search?topic=rag
```

Be careful:

Do not use query parameters for large text prompts or long context. Use a POST request body.

## Request Body

Syntax:

```python
class RequirementRequest(BaseModel):
    requirement: str

@app.post("/generate-test-case")
def generate_test_case(request: RequirementRequest):
    return {"requirement": request.requirement}
```

Meaning:

Defines the JSON data the client must send.

When to use:

Use when input is structured or too large for a URL.

Example:

```json
{"requirement": "User can login"}
```

Be careful:

Wrong field names cause validation errors.

## Response Body

Syntax:

```python
return {"status": "ok"}
```

Meaning:

Returns data from the server to the client.

When to use:

Use in every endpoint to send a useful result.

Example response:

```json
{"status":"ok"}
```

Be careful:

`print()` writes to the terminal. `return` sends the API response.

## Pydantic Model

Syntax:

```python
class ContextQuestionRequest(BaseModel):
    question: str
    context: str
```

Meaning:

Defines and validates the expected data shape.

When to use:

Use for POST request bodies.

Example:

```json
{
  "question": "What is FastAPI?",
  "context": "FastAPI builds APIs."
}
```

Be careful:

`BaseModel` is required for Pydantic validation behavior.

## Validation Error

Example:

```text
422 Unprocessable Entity
```

Meaning:

The request did not match the expected type or shape.

When to use:

Read it when Swagger UI or a client request fails.

Example cause:

```json
{"wrong": "hello"}
```

when the model expects:

```json
{"text": "hello"}
```

Be careful:

Do not ignore the `detail` field. It usually tells you the exact missing or invalid field.
