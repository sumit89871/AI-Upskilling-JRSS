# 23 Mettl Screening Prep

## What this module is

This module prepares you for Mettl-style technical screening questions for the AI Engineer JRSS reskilling path.

The focus is not memorizing answers blindly. The focus is understanding common question patterns, traps, and the reason behind each answer.

## What Mettl-style questions usually test

Mettl-style screening usually checks whether you understand:

- basic Python syntax and output
- data structures
- function behavior
- exception behavior
- REST API fundamentals
- FastAPI request/response basics
- Git command purpose
- Docker and Kubernetes vocabulary
- LLM and prompt engineering fundamentals
- RAG, MCP, and LangGraph mental models

## How each question is written

MCQs include:

- question
- options
- correct answer
- beginner-friendly explanation
- why other options are wrong when useful
- common trap

Coding/output questions include:

- code or problem
- expected answer
- explanation
- common mistake

## Practical exam strategy

Read the question slowly. Many traps are based on one word:

- "staged" vs "committed"
- "container" vs "image"
- "pod" vs "deployment"
- "prompt" vs "completion"
- "RAG" vs "fine-tuning"
- `print()` vs `return`
- list mutation vs new list creation

If you are unsure, eliminate wrong options first. Mettl questions often include one clearly wrong definition, one partially true answer, and one precise answer.
